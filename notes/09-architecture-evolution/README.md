# 09 · 软件架构演化与维护（教材第 10 章）

## 核心考点

- 演化驱动：需求变化、技术债务、性能瓶颈、合规要求
- 演化方式：重构（Refactoring）、重写（Rewrite）、改造（Re-architect）
- 架构腐化（Architecture Erosion）识别
- 软件维护类型：**改正性 / 适应性 / 完善性 / 预防性**
- 演化评估：代价 vs 收益、风险评估

## 常考论文方向

- 单体拆分为微服务（云原生改造）
- 遗留系统迁移上云
