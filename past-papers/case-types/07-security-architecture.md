# 案例题型 07 · 安全架构分析

> 中频题型，近 5 年偶尔出现，25 分。

## 考点分布

- 识别**安全威胁**（STRIDE）
- 设计**纵深防御**方案（分层）
- **加密**算法选择（对称 / 非对称 / Hash）
- **认证 + 授权**（SSO / MFA / RBAC / ABAC）
- 合规要求（**等保 2.0 / GDPR**）

## 核心理论（必背）

### CIA 三要素

- **C** onfidentiality 保密性
- **I** ntegrity 完整性
- **A** vailability 可用性

### STRIDE 威胁建模

| 威胁 | 含义 | 对策 |
|---|---|---|
| **S**poofing | 身份伪造 | 认证、MFA、证书 |
| **T**ampering | 数据篡改 | HMAC、数字签名 |
| **R**epudiation | 抵赖 | 数字签名、审计日志 |
| **I**nformation Disclosure | 信息泄露 | 加密（TLS/AES）、脱敏 |
| **D**oS | 拒绝服务 | 限流、WAF、DDoS 清洗 |
| **E**levation of Privilege | 权限提升 | 最小权限、RBAC、沙箱 |

### 纵深防御 5 层

1. **网络**：防火墙、WAF、IPS/IDS、DDoS 清洗
2. **主机**：HIDS、基线加固、漏洞扫描
3. **应用**：认证授权、输入校验、CSP、SQL 注入防护
4. **数据**：加密（传输/存储）、脱敏、DLP、KMS
5. **管理 / 审计**：日志审计、SIEM、应急响应

### 加密算法

| 类型 | 特点 | 代表 | 国密 |
|---|---|---|---|
| 对称 | 快、共享密钥 | AES / 3DES | **SM4** |
| 非对称 | 慢、公私钥 | RSA / ECC | **SM2** |
| Hash | 不可逆 | SHA-256 / MD5 | **SM3** |
| 签名 | 私钥签公钥验 | RSA / ECDSA | SM2 |

### 认证与授权

- **认证**（Authentication）：**你是谁**—— 密码、MFA、生物、证书
- **授权**（Authorization）：**你能干啥**—— RBAC / ABAC / ACL
- **SSO**：CAS / OAuth2 / OIDC / SAML
- **MFA**：短信 / TOTP / U2F / 生物

### 等保 2.0

- **5 级**分类（一般系统**三级**起）
- **技术**5 项：物理、网络、边界、计算环境、管理中心
- **管理**5 项：安全管理制度、机构、人员、建设、运维

## 答题模板

### 问题 1：识别威胁（STRIDE 分析）

**套路**：
> 针对系统的【登录 / 支付 / 管理后台 / 数据同步】模块，识别：
> - **S 身份伪造**：弱密码，对策 **MFA + 密码复杂度**
> - **T 数据篡改**：中间人攻击，对策 **TLS 1.3 + HMAC**
> - **I 信息泄露**：日志含敏感，对策 **脱敏 + 访问控制**
> - **D DoS**：暴刷接口，对策 **限流 + 滑动窗口**

### 问题 2：设计纵深防御

**五层标准答案**：
```
网络层：防火墙 + WAF + DDoS 清洗
主机层：HIDS + 基线加固 + 漏扫周期 7 天
应用层：SSO + MFA + RBAC + 输入校验 + CSRF Token
数据层：TLS 1.3（传输）+ AES-256（存储）+ 字段脱敏 + KMS
审计层：全链路日志 + SIEM + 100% 关键操作审计
```

### 问题 3：加密方案

**套路**：
- **传输**：TLS 1.3（强制）
- **存储密码**：**Bcrypt / Scrypt / Argon2**（加盐 + 慢哈希）
- **敏感字段**：**AES-256-GCM**（带认证）
- **密钥**：**KMS 托管 + 定期轮换 90d**
- **身份证 / 手机**：**国密 SM4**（合规要求）

### 问题 4：合规分析

**套路**：
> 本系统属于**等保三级**，需满足：
> - 身份鉴别：**双因素认证**
> - 访问控制：**最小权限 + 职责分离**
> - 安全审计：**100% 关键操作日志 + 6 个月保留**
> - 数据保护：**传输加密 + 存储加密 + 备份加密**
> - 入侵防范：**HIDS + NIDS + 漏洞扫描**

## 万能高分句

- "依据 **STRIDE 威胁建模**，识别出 **6 类威胁**，逐一设计对策"
- "采用**纵深防御**策略，从**网络 / 主机 / 应用 / 数据 / 审计**五层构建防护"
- "核心敏感数据采用 **AES-256-GCM + KMS 密钥轮换**，满足**等保三级**"
- "实施**最小权限 + 职责分离**，基于 **RBAC** 精细化到**菜单 / 按钮 / 字段**级"

## 常见陷阱

| ❌ | ✅ |
|---|---|
| 只讲 HTTPS | 分层防御，**5 层都要覆盖** |
| MD5 存密码 | 必须 **Bcrypt/Scrypt/Argon2** |
| 对称加密公网传 | 对称**配合非对称**交换密钥 |
| 无审计 | 日志 + SIEM **必写** |
| 忽视合规 | **等保 / GDPR** 必须提 |
