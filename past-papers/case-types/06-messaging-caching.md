# 案例题型 06 · 消息中间件 / 缓存应用

> 高频题型，近 5 年出现多次，25 分。

## 考点分布

- 消息队列**选型对比**（Kafka / RabbitMQ / RocketMQ）
- 保证**可靠性**（不丢、不重、顺序）
- 缓存**三大问题**（穿透、击穿、雪崩）
- 缓存一致性
- 热点 Key / 大 Key 问题

## 消息队列对比（必背）

| 维度 | Kafka | RabbitMQ | RocketMQ | Pulsar |
|---|---|---|---|---|
| 吞吐 | **百万级** | 万级 | 十万级 | 百万级 |
| 时延 | ms 级 | **μs 级** | ms 级 | ms 级 |
| 可靠 | 高（副本） | 高 | **金融级** | 高 |
| 功能 | 流式为主 | **协议丰富**（AMQP） | 事务消息 / 顺序 | 存算分离 |
| 语言 | Scala | Erlang | Java | Java |
| 适用 | 日志 / 大数据 | 业务消息 | **金融交易** | 云原生 |

## 消息可靠性三要素

### 1. 不丢消息

- **生产端**：同步发送 + 确认 ACK + 重试
- **Broker**：**多副本 + 持久化**（Kafka min.insync.replicas ≥ 2）
- **消费端**：**手动 ACK**，处理成功后再提交 offset

### 2. 不重复消费

- **幂等**：业务去重（唯一键 / Redis setnx / 状态机）
- **消费端去重表**

### 3. 顺序消息

- **全局顺序**：单分区（吞吐低）
- **局部顺序**：按业务主键 hash 到同一分区（推荐）

## 缓存三大问题（必背）

### 1. 缓存穿透（**不存在的数据**）

- **布隆过滤器**（Bloom Filter）
- **缓存空值**（短 TTL）

### 2. 缓存击穿（**热点过期瞬间**）

- **互斥锁**（Redis setnx）
- **永不过期** + 异步更新
- **预热**

### 3. 缓存雪崩（**大量同时过期**）

- **过期时间加随机**（TTL + rand）
- **多级缓存**（本地 Caffeine + Redis）
- **熔断降级**

## 缓存一致性（Cache Aside 模式）

**推荐：先更新 DB，再删除缓存**

| 方案 | 问题 |
|---|---|
| 先删缓存再更 DB | 并发下读请求会把旧值写回缓存 |
| **先更 DB 再删缓存** ✅ | 极小窗口期可能读到旧值（可接受） |
| 延迟双删 | 更新前删 + sleep + 再删 |
| 订阅 binlog | Canal 监听变更异步失效缓存（**强一致方案**） |

## 答题模板

### 问题 1：消息队列选型

**标准回答**：
> 本系统为**金融支付**场景，要求：
> - **不丢消息**（金融级可靠）
> - 十万级 TPS
> - 事务消息、顺序消息
>
> 综合对比，选 **RocketMQ**：
> - 同步双写 + **SYNC_MASTER** 零丢失
> - 内建**事务消息**支持半消息回查
> - 阿里生产级验证

### 问题 2：可靠性设计

**模板三段式**：
> - 生产端：**同步发送 + confirm + 失败重试 3 次**
> - Broker：**主从同步复制 + 持久化**
> - 消费端：**手动 ACK + 本地幂等表**
> 实现 **at-least-once + 业务幂等 = 有且仅一次**的效果。

### 问题 3：缓存方案设计

**模板**：
```
1. 读：Cache Aside，缓存命中直接返回，未命中回源 DB 后写缓存
2. 写：先更新 DB，再删除缓存
3. 防穿透：布隆过滤器前置
4. 防击穿：热点 Key 永不过期 + 异步刷新
5. 防雪崩：TTL = base + random(0, 300s)
6. 最终一致：Canal 监听 binlog 兜底失效缓存
```

## 万能高分句

- "选型 **RocketMQ**，同步双写 + 手动 ACK + 本地幂等表实现 **exactly-once**"
- "通过 **布隆过滤器 + 缓存空值** 两级防穿透，将数据库压力降低 **99%**"
- "热点 Key **永不过期 + 异步刷新**，避免击穿瞬间数据库雪崩"
- "缓存一致性采用 **Cache Aside（先更 DB 后删缓存）**，辅以 **Canal 监听 binlog** 兜底"

## 常见陷阱

| ❌ | ✅ |
|---|---|
| 选型不给理由 | 必须对比 **2–3 种** 说明差异 |
| 只说不丢 | **不丢、不重、顺序**三要素都要讲 |
| 穿透击穿混淆 | **穿透 = 不存在**；**击穿 = 热点过期** |
| 先删缓存再更 DB | **推荐反过来**（并发安全） |
| 无多级缓存 | 加 **本地 + 分布式** 两级 |
