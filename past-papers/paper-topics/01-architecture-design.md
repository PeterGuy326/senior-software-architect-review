# 论文主题 01 · 软件架构设计（通用）

## 历年出题角度（公开资料整理）

- 论软件架构风格及其应用
- 论 ABSD（基于架构的软件开发）
- 论软件架构的风格选择与决策
- 论软件架构的多视图建模

## 一、核心理论（400 字以内）

### 1. 软件架构定义

软件架构（Software Architecture）是系统的**基本结构**，包括软件元素、元素之间的关系、以及二者的特性。架构为软件系统提供了结构、行为和属性的高级抽象。

### 2. 主流架构风格

- **数据流**：批处理、管道—过滤器
- **调用—返回**：主程序—子程序、OO、层次式、C/S
- **独立构件**：进程通信、事件驱动（发布—订阅）
- **虚拟机**：解释器、规则系统
- **数据中心**：数据库仓库、黑板
- **闭环控制**：过程控制

### 3. 4+1 视图模型

逻辑视图、开发视图、进程视图、物理视图 + 场景视图

### 4. ABSD 过程（6 步）

架构需求 → 架构设计 → 文档化 → 复审 → 实现 → 演化
