# 论文主题 08 · 基于构件的软件开发（CBSD）

## 历年出题角度

- 论基于构件的软件开发及其应用
- 论软件复用技术的应用
- 论构件组装与集成

## 一、核心理论

### CBSD 定义

**基于构件的软件开发**（Component-Based Software Development）通过**组装预制构件**构建系统，强调**复用**。

### 构件的要素

- **接口契约**（规约）
- **独立部署**
- **组合能力**
- **无持久状态**（或明确说明）

### 主流构件标准

| 标准 | 出身 | 特点 |
|---|---|---|
| **CORBA** | OMG | 跨语言、重量级 |
| **COM/DCOM** | 微软 | Windows 生态 |
| **EJB** | Sun/Java | Java EE |
| **.NET Component** | 微软 | .NET |
| **OSGi** | — | 模块化 Java |

### 构件复用层次

1. **代码复用**（函数、类库）
2. **设计复用**（模式、框架）
3. **分析复用**（领域模型）
4. **架构复用**（参考架构）

### CBSD 开发过程

1. 构件获取（开发 / 购买 / 开源）
2. 构件管理（构件库）
3. 构件检索（描述、分类）
4. 构件组装（粘合代码）
5. 构件演化（版本、替换）

## 三、关键数据

- 构件数：**50+ 通用构件**
- 复用率：**50–70%**
- 交付周期：**缩短 40–60%**

## 四、万能句式

- "构件以 **OpenAPI 契约**为中心，通过 **Nexus** 统一托管"
- "抽取 **用户 / 权限 / 支付 / 订单** 4 大通用业务构件，复用率 **60%+**"
- "采用 **OSGi** 实现构件**热插拔**，业务变更无需重启"

## 五、避坑点

| ❌ | ✅ |
|---|---|
| 构件 = 类 | 构件是 **可独立部署的单元** |
| 无接口契约 | 契约是构件**可替换**的基础 |
| 只说开发 | **构件库 + 治理 + 演化**全流程 |
| 与微服务混淆 | 构件偏 **部署粒度**，微服务偏 **服务自治** |
