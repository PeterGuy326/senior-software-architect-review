# 论文主题 12 · 软件测试与质量保证

## 历年出题角度

- 论软件测试方法与策略
- 论软件质量保证体系
- 论自动化测试

## 一、核心理论

### 测试分类（按层级）

| 层级 | 测试 | 代表工具 |
|---|---|---|
| 单元测试 | 函数 / 类 | JUnit / pytest |
| 集成测试 | 模块间 | TestNG |
| 系统测试 | 整体 | — |
| 验收测试 | 用户视角 | Selenium |

### 测试金字塔（Mike Cohn）

```
       /\  UI
      /  \ （少量）
     /----\
    / 集成 \（中等）
   /--------\
  /  单元    \（大量）⭐
 /------------\
```

### 测试方法

- **黑盒**：等价类、边界值、因果图、判定表、场景法
- **白盒**：语句 / 判定 / 条件 / 路径覆盖
- **灰盒**：结合

### 非功能测试

- **性能**（JMeter / Gatling）
- **安全**（SonarQube / OWASP ZAP）
- **兼容性**（BrowserStack）
- **可用性**（用户体验测试）
- **混沌工程**（ChaosBlade / Chaos Monkey）

### CI/CD 测试左移右移

- **左移**：开发早期引入测试（TDD/BDD、静态扫描）
- **右移**：生产监控、A/B、全链路压测

### 质量保证体系

- **过程**：CMMI / ISO 9001
- **标准**：ISO 25010（8 质量特性）
- **度量**：缺陷密度、覆盖率、逃逸率

## 二、万能提纲

```
1. 背景（350 字）
   - 系统质量事故频发，用户投诉高
2. 理论（350 字）
   - 测试金字塔、黑白盒、左移右移、CMMI
3. 实践论述（1600 字）⭐⭐⭐⭐⭐
   (1) 单元测试：JUnit + 覆盖率 80%（Jacoco）
   (2) 接口测试：Postman / RestAssured
   (3) UI 自动化：Selenium + 页面对象模型
   (4) 性能测试：JMeter 全链路压测
   (5) 安全测试：SAST（SonarQube）+ DAST（ZAP）
   (6) 混沌工程：Chaos Mesh 注入故障
   (7) CI/CD：Jenkins Pipeline 质量门禁
4. 成效（250 字）
   - 缺陷逃逸率 10% → 1%，发布频率提升 10 倍
```

## 三、关键数据

- 单元覆盖率：**> 80%**
- 缺陷密度：**< 0.5 / KLOC**
- 缺陷逃逸率：**< 1%**
- 自动化覆盖：**回归用例 80%**

## 四、万能句式

- "遵循**测试金字塔**，单元测试占 70%，接口 20%，UI 10%"
- "实施**质量左移**，开发阶段 **SAST + 单测** 提前拦截"
- "通过**全链路压测**验证大促容量，发现并修复 X 个性能瓶颈"

## 五、避坑点

| ❌ | ✅ |
|---|---|
| 只讲测试类型 | **金字塔 + 左移右移**体现策略 |
| 无覆盖率数据 | 覆盖率、逃逸率必须量化 |
| 忽视性能/安全 | **非功能测试**必写 |
| 无质量门禁 | CI 卡单测 / 扫描结果 |
