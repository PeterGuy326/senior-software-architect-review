# 论文主题 02 · 软件架构评估（ATAM）

## 历年出题角度

- 论软件系统架构评估及其应用（**高频**）
- 论质量属性驱动的架构评估
- 论 ATAM 在 XX 系统中的应用

## 一、核心理论（必写）

### 三大评估方法对比

| 方法 | 特点 | 适用 |
|---|---|---|
| **SAAM** | 基于场景，最早 | 早期、可修改性为主 |
| **ATAM** | **关注权衡，最主流** | 设计阶段 |
| **CBAM** | ATAM + 经济学成本分析 | 投资决策 |

### ATAM 四类关键点（**论文必写**）

- **敏感点**：为实现某质量属性所关键的架构决策
- **权衡点**：同时影响多个质量属性（有正有负）
- **风险点**：可能导致风险的决策
- **非风险点**：已确认安全的决策

### ATAM 9 步流程

1. 介绍 ATAM → 2. 业务驱动 → 3. 被评估架构 → 4. 识别架构方法 → 5. 质量属性**效用树**
6. 分析架构方法 → 7. 场景集 + 优先级 → 8. 分析架构方法（迭代） → 9. 呈现结果

### 质量属性效用树

```
Utility（根）
├── 性能
│   ├── 数据延迟 (H,M)
│   └── 事务吞吐 (M,H)
├── 可用性
│   └── 硬件故障 (H,L)
└── 可修改性
    └── 新增功能 (L,M)
```

优先级：**(业务价值, 技术难度)**

## 二、万能提纲

```
1. 项目背景 + 为何需要评估（300 字）
2. ATAM 理论（400 字）
3. 实践论述（1500+ 字）
   (1) 业务驱动识别（利益相关者 + 质量目标）
   (2) 质量属性效用树构建
   (3) 场景生成（6 元组）

   (5) 提出改进方案 + 验证
4. 评估成效 + 不足（250 字）
```

## 三、实战示例（答题素材）

### 典型敏感点举例

- 主备切换 < 30s → 影响**可用性**
- 缓存命中率 > 95% → 影响**性能**

### 典型权衡点举例

- 全链路 HTTPS 加密 → **+安全 / -性能**
- 数据异步写入 → **+性能 / -一致性**

### 典型风险点举例

- 单点服务无熔断 → 级联故障
- MQ 未持久化 → 宕机丢消息

### 典型非风险点

- 采用业界成熟的 Kafka
- 数据库主从复制

## 四、万能句式

- "通过 ATAM 方法，识别出 **3 个敏感点、2 个权衡点、4 个风险点**"
- "针对风险点【XX】，我们引入了【战术】，将风险转为非风险"
- "效用树中【性能-吞吐量 (H,H)】是最高优先级，因此在架构决策中优先保证"

## 五、避坑点

| ❌ | ✅ |
|---|---|
| 只写 ATAM 流程，无项目结合 | **场景化**：给出具体决策 + 四类点判别 |
| 四类点混淆 | 用"为了…" / "既…又…" / "可能…" / "已验证…"信号词区分 |
| 无量化 | 每个敏感点写清**响应度量** |
