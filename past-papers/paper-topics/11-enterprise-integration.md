# 论文主题 11 · 企业应用集成（EAI）

## 历年出题角度

- 论企业应用集成（EAI）技术
- 论异构系统集成方案
- 论数据集成与交换

## 一、核心理论

### EAI 四个层次

1. **表示集成**（Portal、界面集成）
2. **数据集成**（ETL、数据仓库）
3. **控制集成 / 应用集成**（API、ESB）
4. **业务流程集成**（BPM、工作流）

### 集成模式（Hohpe）

| 模式 | 特点 |
|---|---|
| **文件传输** | 简单、异步、粒度粗 |
| **共享数据库** | 紧耦合 |
| **远程调用**（RPC） | 同步、耦合 |
| **消息传递**（MQ） | **解耦、异步、推荐** |

### ETL vs ELT

- **ETL**：传统数仓，转换在入库前
- **ELT**：大数据时代，原始入湖后再处理

### 主流技术

- **消息中间件**：Kafka / RabbitMQ / RocketMQ
- **ESB**：MuleSoft / WSO2
- **工作流 / BPM**：Activiti / Camunda
- **数据集成**：Kettle / DataX / Flink CDC

- 集成系统数：**20+**

## 四、万能句式

- "采用 **消息传递** 模式解耦（Kafka），避免 RPC 强依赖"

## 五、避坑点

| ❌ | ✅ |
|---|---|
| 只讲 ETL | EAI **四层全** |
| 点对点集成 | 走**总线 / 网关** |
| 无主数据治理 | MDM 是**集成成败关键** |
| 缺监控 | 跨系统链路必须**全链路追踪** |
