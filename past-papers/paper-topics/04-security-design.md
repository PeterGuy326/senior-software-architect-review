# 论文主题 04 · 系统安全性与保密性设计

## 历年出题角度

- 论系统安全架构设计及其应用（**高频**）
- 论软件系统中的信息安全
- 论零信任架构在 XX 系统的应用

## 一、核心理论

### CIA 三要素

- **C**onfidentiality 保密性
- **I**ntegrity 完整性
- **A**vailability 可用性

### 纵深防御 5 层

1. 网络层
2. 主机层
3. 应用层
4. 数据层
5. 管理 / 审计层

### STRIDE 威胁建模

| 威胁 | 含义 | 对策 |
|---|---|---|
| **S**poofing | 身份伪造 | 认证（SSO + MFA） |
| **T**ampering | 数据篡改 | 完整性校验（HMAC） |
| **R**epudiation | 抵赖 | 日志 + 数字签名 |
| **I**nformation Disclosure | 信息泄露 | 加密（TLS + AES） |
| **D**oS | 拒绝服务 | 限流 + DDoS 清洗 |
| **E**levation of Privilege | 权限提升 | 最小权限 + RBAC |

### 密码学基础

- 对称：DES / 3DES / **AES** / 国密 **SM4**
- 非对称：**RSA** / ECC / 国密 **SM2**
- Hash：MD5 / SHA-256 / 国密 **SM3**
- 数字签名：私钥签 + 公钥验

### 等保 2.0

- 五级分类（一般系统**三级**起）
- 10 大技术要求（物理 / 网络 / 边界 / 计算环境 / 管理中心 + 5 项管理）

## 二、万能提纲

```
1. 项目背景（350 字）
   - 【金融 / 政务 / 医疗】系统，等保三级，百万级敏感数据
2. 理论（350 字）
   - CIA、STRIDE、纵深防御、等保
3. 实践论述（1600 字）⭐⭐⭐⭐⭐
   (1) 网络层：防火墙 + WAF + DDoS 清洗
   (2) 主机层：HIDS + 基线加固 + 漏扫
   (3) 应用层：SSO + MFA + RBAC + 输入校验 + CSP
   (4) 数据层：TLS 1.3 传输 + AES-256 存储 + 字段脱敏 + DLP
   (5) 审计：全链路日志 + SIEM + 合规报告
4. 成效与展望（250 字）
   - 通过等保三级测评
   - 零重大安全事件
```

## 三、实战数据

- 等保等级：**三级**（政企标配）
- 加密算法：AES-256、TLS 1.3、SM2/3/4
- 认证强度：**SSO + MFA**
- 审计覆盖：**100% 关键操作**
- 漏洞修复 SLA：高危 **24h**，中危 **7 天**

## 四、万能句式

- "针对等保三级合规要求，采用纵深防御策略，从**网络、主机、应用、数据、审计**五层建立防护"
- "通过 **STRIDE 威胁建模**识别出 X 类高危威胁，逐一设计对策"
- "关键数据采用 **AES-256 + 密钥管理 KMS**，传输层 TLS 1.3"
- "实施**最小权限 + 职责分离**，通过 **RBAC** 控制访问"

## 五、避坑点

| ❌ | ✅ |
|---|---|
| 只讲加密算法 | **分层防御**体现架构思维 |
| 忽视合规 | **等保 / GDPR / SOX** 明确写 |
| 无审计追溯 | 审计日志是监管重点 |
| 方案一刀切 | **数据分级**，敏感级别不同方案不同 |
