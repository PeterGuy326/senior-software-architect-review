# 论文主题 07 · 面向服务架构（SOA）

## 历年出题角度

- 论面向服务的架构设计及其应用
- 论 SOA 与微服务的演进
- 论企业服务总线（ESB）的应用

## 一、核心理论

### SOA 核心原则

1. **服务复用**
2. **松耦合**
3. **契约标准化**
4. **服务自治**
5. **无状态**
6. **可组合**
7. **可发现**

### SOA 与微服务对比

| 维度 | SOA | 微服务 |
|---|---|---|
| 粒度 | 粗（企业级） | 细（业务功能） |
| 通信 | **ESB**（SOAP/WS-*) | 轻量（REST/gRPC） |
| 数据 | 共享数据库多见 | Database Per Service |
| 治理 | 集中 | 去中心化 |
| 部署 | 单体部署多 | 独立部署 |

### SOA 参考模型

- **服务消费者** ⇄ **服务注册中心**（UDDI）⇄ **服务提供者**
- 通过 **WSDL** 描述，**SOAP** 传输，**ESB** 编排

### Web Service 三件套

- **SOAP**：协议（基于 XML）
- **WSDL**：服务描述
- **UDDI**：服务注册/发现

## 二、万能提纲（传统企业系统 SOA 化）

```
1. 背景（350 字）
   - 【银行 / 运营商】异构系统烟囱林立，集成困难
2. 理论（350 字）
   - SOA 原则、与微服务对比、ESB 角色
3. 实践论述（1600 字）⭐⭐⭐⭐⭐
   (1) 服务识别：业务流程自顶向下拆分
   (2) 服务设计：WSDL 契约先行
   (3) ESB 集成：MuleSoft / IBM ESB / WSO2
   (4) 服务注册：UDDI / 或 API Gateway
   (5) 治理：BPEL 编排 + 监控 + SLA
   (6) 安全：WS-Security + OAuth2
4. 成效（250 字）
   - 集成成本降低 60%，新业务上线周期 6m → 1m
```

## 三、关键数据

- 服务数：**30–100 个企业级服务**
- 复用率：**50%+**
- 上线周期：**缩短 80%**

## 四、万能句式

- "通过 **ESB** 屏蔽异构系统差异，以 **WSDL** 契约规范服务接口"
- "基于 SOA 做**服务目录**，提升跨部门**服务复用率至 50%+**"
- "**演进路线**：单体 → SOA（ESB）→ 微服务（去中心化）"

## 五、避坑点

| ❌ | ✅ |
|---|---|
| SOA = 微服务 | 必须讲 **区别与联系** |
| 不提 ESB | ESB 是 SOA 核心组件 |
| 丢契约 | **WSDL / OpenAPI** 契约是 SOA 根基 |
| 无演进观 | 体现 **SOA → 微服务** 的技术演化 |
