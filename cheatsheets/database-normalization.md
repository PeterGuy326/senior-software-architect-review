# 数据库范式速查

## 范式层级（必背）

| 范式 | 条件 | 解决问题 |
|---|---|---|
| **1NF** | 属性不可再分（原子性） | 消除重复组 |
| **2NF** | 1NF + 非主属性完全依赖候选码 | 消除**部分函数依赖** |
| **3NF** | 2NF + 非主属性不传递依赖候选码 | 消除**传递函数依赖** |
| **BCNF** | 3NF + 主属性也不存在部分和传递依赖 | 消除主属性依赖异常 |
| **4NF** | BCNF + 消除多值依赖 | 独立的多值属性分离 |

## 判定流程

```
1. 找候选码（从左部找，逐步扩展闭包）
2. 判 1NF：原子值？
3. 判 2NF：非主属性是否完全依赖候选码？（有组合码才可能违反）
4. 判 3NF：非主属性是否存在 A → B → C 的传递依赖？
5. 判 BCNF：所有函数依赖 X → Y，X 必须是超码
```

## 函数依赖核心概念

| 术语 | 含义 |
|---|---|
| **函数依赖** X → Y | X 决定 Y |
| **完全依赖** | Y 依赖于 X 且不依赖于 X 的任何真子集 |
| **部分依赖** | Y 依赖于 X 的某个真子集 |
| **传递依赖** | X → Y, Y → Z (Y ↛ X) 则 X → Z 是传递 |
| **平凡函数依赖** | Y ⊆ X |

## Armstrong 公理

- **自反律**：Y ⊆ X ⇒ X → Y
- **增广律**：X → Y ⇒ XZ → YZ
- **传递律**：X → Y, Y → Z ⇒ X → Z

推论：合并、分解、伪传递

## 候选码求解步骤

1. 将只在左边出现的属性组 L 求闭包 L⁺
2. 若 L⁺ 覆盖全部属性 → L 即候选码
3. 否则从左右都出现的属性中加入试探

## 分解原则

- **无损连接**：自然连接后不丢失信息
- **保持依赖**：分解后仍能推出原函数依赖
- 3NF 分解可以**同时**保持无损 + 保持依赖
- BCNF 分解保证无损，但**可能破坏依赖保持**

## 案例题答题套路

> 给 R(A,B,C,D,E) 和 FD 集 F，要求：判断范式 + 分解到 3NF

1. 求候选码
2. 主/非主属性划分
3. 逐项判 2NF、3NF
4. 若不满足，按 FD 分解为多个关系模式
5. 验证：无损连接 + 依赖保持
