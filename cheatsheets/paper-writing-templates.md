# 论文写作模板（🔥 4 类必备）

## 总体要求

- **字数**：≥ 2500（摘要 300 左右）
- **时间**：120 分钟
- **合格线**：45 分
- **结构**：摘要 + 正文三段式
  - 摘要：项目背景 + 自己角色 + 项目成果（300 字）
  - 正文：
    1. **项目背景**（300–400 字）—— 项目规模、团队、时间、业务价值
    2. **理论阐述**（400–500 字）—— 本次题目所需的理论知识
    3. **实践论述**（1500+ 字）—— **重点**，围绕题目要求展开，至少 3 点
    4. **总结与展望**（200 字）—— 成效 + 不足 + 改进

## 通用写作套路（5 步法）

1. **选题**：看完 4 题，选**自己项目最贴合**的（别硬凑）
2. **摘要**：先写（300 字），定下基调
3. **列提纲**：正文三级目录，每级一句话
4. **量化**：必带数据（QPS、RT、可用性、成本 X%）
5. **收尾**：承认不足（显得真实） + 改进方向

## 四类必备模板

### 模板 1 · 软件架构设计（通用）

**常见题目关键词**：质量驱动、架构风格选择、架构权衡

**提纲**：
1. 项目背景（XX 系统，用户规模、QPS）
2. 理论：架构风格 / 质量属性 / ATAM
3. 实践：
   - (1) 识别关键质量属性（可用性、性能、可扩展）
   - (2) 架构风格选型（如选择微服务 + 事件驱动的**理由**）
   - (3) 权衡分析（ATAM 四类点举例）
4. 总结：效果数据 + 改进

### 模板 2 · 系统可靠性分析与设计

**提纲**：
1. 项目背景（业务对可用性要求，如金融/通信 99.99%）
2. 理论：MTBF / MTTR、冗余、容错（N 版本、恢复块）
3. 实践（**3–4 点**）：
   - (1) 故障检测：心跳、健康检查、监控告警
   - (2) 冗余设计：主备/多活、数据多副本
   - (3) 快速恢复：自动切换、降级、熔断
   - (4) 容量保护：限流、隔离（舱壁模式）
4. 总结：可用性从 99.5% → 99.95%

### 模板 3 · 系统安全性与保密性设计

**提纲**：
1. 项目背景（涉及敏感数据，合规要求等保三级）
2. 理论：CIA 三要素、纵深防御、零信任、STRIDE
3. 实践（**分层展开**）：
   - (1) 网络层：防火墙、WAF、DDoS 清洗
   - (2) 主机层：HIDS、漏洞扫描、基线加固
   - (3) 应用层：认证（SSO + MFA）、授权（RBAC）、输入校验
   - (4) 数据层：加密（传输 TLS、存储 AES）、脱敏、DLP
   - (5) 审计：全链路日志、合规报告
4. 总结：零重大安全事件 + 通过等保测评

### 模板 4 · 系统建模（UML / DSSA）

**提纲**：
1. 项目背景
2. 理论：UML 4+1 视图、DSSA 三层/三角色
3. 实践：
   - (1) 用例视图：用例图 + 参与者
   - (2) 逻辑视图：类图 / 组件图
   - (3) 进程视图：顺序图 / 活动图
   - (4) 物理视图：部署图
4. 总结

## 扩展备选模板（有余力再准备）

- 云原生改造（单体 → 微服务）
- 分布式系统设计（CAP、一致性方案）
- 中间件选型（消息队列对比）
- 数据架构设计（Lambda / Kappa）

## 写作禁忌（避雷）

| ❌ 不要 | ✅ 要 |
|---|---|
| 通篇理论无实践 | 理论 1/4，实践 3/4 |
| 没有项目背景 | 开头 300 字交代清楚 |
| 数据空泛（"大幅提升"） | 量化（"从 X 提升到 Y"） |
| 一碗水端平 | 突出 2–3 个亮点 |
| 全部成功 | 承认 1 个问题 + 改进 |
| 字数不够 | 至少 2500，最好 2800 左右 |

## 手写训练建议

1. 按 4 类模板**各手写 1 遍**，计时 120 分钟
2. 准备 1 个**万能项目**，覆盖 3–4 类主题
3. 考前 2 周开始，每周 1 篇
