# 软件度量速查（📚 综合知识 1–2 题）

## 一、规模度量

### 代码行 LOC

- 总代码行、有效代码行、注释行
- 简单但语言依赖

### 功能点 FP（Function Point）

**5 类用户功能**：

1. **内部逻辑文件**（ILF）
2. **外部接口文件**（EIF）
3. **外部输入**（EI）
4. **外部输出**（EO）
5. **外部查询**（EQ）

### UFP 未调整功能点

UFP = Σ(每类项数 × 权重)

### 调整功能点 AFP

AFP = UFP × (0.65 + 0.01 × TCF)

## 二、复杂度度量

### McCabe 圈复杂度（V(G)）

**公式**（任选其一）：

- V(G) = 边数 - 节点数 + 2
- V(G) = 判定节点数 + 1
- V(G) = 封闭区域数 + 1

**判定标准**：

| V(G) | 评价 |
|---|---|
| 1–10 | 简单、易测试 |
| 11–20 | 中等复杂 |
| 21–50 | 复杂 |
| > 50 | 不可测，必须重构 |

## 三、质量度量

### 缺陷密度

缺陷密度 = 缺陷数 / KLOC（千行代码）

### 测试覆盖率

- 语句覆盖 < 判定覆盖 < 条件覆盖 < 判定/条件覆盖 < 条件组合覆盖 < 路径覆盖

## 四、耦合与内聚度量

### 耦合从低到高（越低越好）

1. **非直接耦合**：无关系
2. **数据耦合**：传简单参数
3. **标记耦合**：传结构体/对象
4. **控制耦合**：传控制标志
5. **外部耦合**：共享外部环境
6. **公共耦合**：共享全局变量
7. **内容耦合**：直接访问内部（最糟）

### 内聚从低到高（越高越好）

1. 偶然内聚
2. 逻辑内聚
3. 时间内聚
4. 过程内聚
5. 通信内聚
6. 顺序内聚
7. **功能内聚**（最好）

> 口诀：**偶逻时过通顺功**

## 五、效率度量（COCOMO）

COCOMO 模型：

- **基本 COCOMO**：PM = a × (KLOC)^b
- **中级 COCOMO**：加 15 个成本驱动因子
- **详细 COCOMO**：按阶段细化

COCOMO II：

- 应用组合模型 / 早期设计模型 / 后体系结构模型
