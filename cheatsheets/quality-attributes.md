# 质量属性与战术速查（🔥 必背）

## 一、六大质量属性定义

| 属性 | 定义 | 度量 |
|---|---|---|
| **可用性** Availability | 系统在规定时间内可正常使用的能力 | MTBF / (MTBF + MTTR) |
| **性能** Performance | 对事件的响应速度 | 响应时间、吞吐量、TPS/QPS |
| **安全性** Security | 抵御未授权访问的能力（CIA） | 抵御攻击时间、检测延迟 |
| **可修改性** Modifiability | 变更的成本 | 变更工时、影响范围 |
| **可测试性** Testability | 通过测试暴露缺陷的难易程度 | 测试时间、覆盖率 |
| **易用性** Usability | 用户完成任务的效率与满意度 | 学习时间、错误率 |

## 二、质量属性场景 6 元组（背下来）

**刺激源 → 刺激 → 制品 → 环境 → 响应 → 响应度量**

## 三、战术速查（记忆口诀）

### 可用性战术（错误检测 + 恢复 + 预防）

- **错误检测**：Ping/Echo、心跳（Heartbeat）、异常检测
- **错误恢复**：冗余（主动/被动/备用）、投票、事务、回滚
- **错误预防**：移除服务、事务、进程监视器

### 性能战术（资源需求 + 资源管理 + 资源仲裁）

- **资源需求**：减少计算开销、降低事件频率、限制执行时间
- **资源管理**：引入并发、增加资源、维持数据副本
- **资源仲裁**：FIFO、固定优先级、动态优先级、静态调度

### 安全性战术（抵抗 + 检测 + 恢复）

- **抵抗攻击**：认证、授权、加密、完整性维护、暴露限制
- **检测攻击**：入侵检测（IDS）
- **从攻击中恢复**：审计追踪、恢复（同可用性）

### 可修改性战术（局部化 + 防止连锁 + 延迟绑定）

- **局部化修改**：高内聚低耦合、预期变更、通用模块
- **防止连锁反应**：信息隐藏、中介（Adapter/Facade）、接口—实现分离
- **延迟绑定时间**：配置文件、多态、反射、运行时注册

### 可测试性战术

- 输入/输出记录与回放
- 内部监测（Built-in Monitor）
- 接口与实现分离

### 易用性战术

- 运行时用户支持（撤销、取消、聚合、多视图）
- 设计时：接口一致性、用户模型

## 四、常见刺激—响应映射（背这个直接写答题）

| 场景类型 | 典型刺激 | 典型响应度量 |
|---|---|---|
| 可用性 | 组件故障 | 故障恢复时间 < X 秒 |
| 性能 | 并发请求到达 | 响应时间 < X ms，吞吐量 ≥ Y TPS |
| 安全性 | 未授权用户访问 | 识别率 ≥ X%，拦截时间 < Y |
| 可修改性 | 新增业务规则 | 修改时间 < X 人日，影响模块数 < Y |
