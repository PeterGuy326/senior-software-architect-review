# 架构风格对比速查

## 五大分类

### 1. 数据流风格

| 风格 | 特点 | 适用 | 局限 |
|---|---|---|---|
| 批处理序列 | 整批数据顺序处理，构件间全量传递 | 离线数据处理、编译 | 不支持并发、交互差 |
| 管道—过滤器 | 流式处理，过滤器独立，管道连接 | Unix 命令、ETL、编译器前端 | 状态共享困难 |

### 2. 调用—返回风格

| 风格 | 特点 | 适用 |
|---|---|---|
| 主程序—子程序 | 集中控制，过程调用 | 传统结构化程序 |
| 面向对象 | 封装、继承、多态 | 业务系统主流 |
| 层次式 | 单向依赖、每层只能调用下层 | Web、操作系统 |
| 客户端—服务器（C/S） | 请求—响应 | 企业应用、数据库 |

### 3. 独立构件风格

| 风格 | 特点 | 适用 |
|---|---|---|
| 进程通信 | 构件独立，消息传递 | 分布式系统 |
| 事件驱动（隐式调用） | 发布—订阅，解耦 | GUI、消息系统、微服务 |

### 4. 虚拟机风格

| 风格 | 特点 | 适用 |
|---|---|---|
| 解释器 | 虚拟机执行中间码 | JVM、脚本引擎 |
| 规则系统 | 规则 + 工作内存 + 推理引擎 | 专家系统、风控 |

### 5. 以数据为中心（仓库）风格

| 风格 | 特点 | 适用 |
|---|---|---|
| 数据库系统 | 中心化数据 + 独立计算 | 企业信息系统 |
| 超文本系统 | 节点 + 链接 | Web |
| **黑板** | 共享知识源 + 控制策略 | 语音识别、模式识别、AI |

### 6. 闭环控制

- 过程控制：传感器 → 控制器 → 执行器，常用于工业控制

## 速记口诀

> **"管批调 OO 层 CS、进程事件、解释规则、数据超黑、过程闭环"**

## 选型决策要点

| 关注点 | 优先风格 |
|---|---|
| 高并发、松耦合 | 事件驱动、微服务 |
| 可维护性 | 分层、OO |
| 数据密集 | 数据仓库、黑板 |
| 流式处理 | 管道—过滤器 |
| AI / 专家系统 | 规则系统、黑板 |
