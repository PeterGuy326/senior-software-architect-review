# 知识点 14 · ABSD + ADL + 4+1 视图

## 一、核心概念

### ABSD（基于架构的软件设计）6 步

1. **架构需求**
2. **架构设计**
3. **文档化**
4. **架构复审**
5. **架构实现**
6. **架构演化**

### 4+1 视图模型（Kruchten）

```
        逻辑视图（功能、类、接口）
           │
开发视图 ──┼── 进程视图
           │   （并发、同步、性能）
        部署视图
        （物理节点、网络）

        +1 场景视图（用例，串起其他 4 视图）
```

| 视图 | 关注 | 建模元素 |
|---|---|---|
| **逻辑** | 功能分解 | 类图、对象图 |
| **开发** | 代码组织 | 包图、组件图 |
| **进程** | 并发性能 | 时序图、活动图 |
| **物理 / 部署** | 部署拓扑 | 部署图 |
| **场景** | 用例串联 | 用例图、时序图 |

### ADL 架构描述语言

- **构件**（Component）
- **连接件**（Connector）
- **配置**（Configuration）

代表 ADL：**Wright / Rapide / UniCon / Acme / C2**

### ISO/IEC/IEEE 42010 标准

- 定义"**架构描述**"的标准元模型：**利益相关者 → 关注 → 视角 → 视图**

## 三、典型例题

### 📚 选择题 1（4+1 视图）

系统部署在多少台服务器、网络拓扑属于哪个视图？
- A. 逻辑  B. 进程  C. **物理 / 部署**  D. 开发

**答案**：C

### 📚 选择题 2（ADL 要素）

下列**不是** ADL 核心要素的是：
- A. 构件  B. 连接件  C. 配置  D. **数据库**

**答案**：D

### 📚 选择题 3（ABSD 步骤）

ABSD 第 4 步是：
- A. 架构设计  B. 文档化  C. **架构复审**  D. 架构实现

**答案**：C

### 🎯 案例填空：4+1 视图选用

系统架构文档通常选 **2–3 个** 视图重点展开：
- 业务侧 → **逻辑视图 + 场景视图**
- 运维侧 → **部署视图 + 进程视图**
- 开发侧 → **开发视图**

### ✍️ 论文场景

"基于 **ABSD 6 步法**开展架构活动：需求 → 设计 → 文档化 → **ATAM 复审** → 实现 → 演化；其中文档化采用 **4+1 视图模型**，重点展开**逻辑视图（领域建模）**和**部署视图（K8s 拓扑）**。"
