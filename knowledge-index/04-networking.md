# 知识点 04 · 网络

## 一、核心概念

- **OSI 7 层**：物-数-网-传-会-表-应（**物理 / 数据链路 / 网络 / 传输 / 会话 / 表示 / 应用**）
- **TCP/IP 4 层**：网络接口 / 网际（IP）/ 传输（TCP/UDP）/ 应用
- **TCP vs UDP**：TCP 可靠有序三次握手；UDP 无连接、延迟低
- **三次握手**：SYN → SYN+ACK → ACK；**四次挥手**：FIN → ACK → FIN → ACK
- **常用端口**：HTTP 80 / HTTPS 443 / SSH 22 / FTP 21 / SMTP 25 / DNS 53 / MySQL 3306 / Redis 6379
- **子网划分**：/24 = 255.255.255.0 = 256 个地址；**借位** n 位分 2^n 子网
- **路由协议**：RIP（距离向量）/ OSPF（链路状态）/ BGP（自治系统间）

## 三、典型例题

### 📚 选择题 1（OSI 层定位）

TCP 协议属于 OSI 第几层？
- A. 2 层  B. 3 层  C. **4 层**  D. 5 层

**答案**：C（传输层）

### 📚 选择题 2（子网划分）

192.168.1.0/26 可容纳的主机数（扣除网络地址和广播地址）？
- A. 64  B. **62**  C. 30  D. 254

**答案**：B
**解析**：/26 主机位 = 6，地址 2^6 = 64，可用 64 - 2 = **62**

### 📚 选择题 3（端口对应）

DNS 查询默认使用端口？
- A. 21  B. 25  C. **53**  D. 80

**答案**：C

### 🎯 案例填空：HTTPS 工作流程

HTTPS = HTTP + **TLS/SSL**，工作流程：
1. 客户端 ClientHello → 发送支持的加密套件
2. 服务器 ServerHello + **证书**（含公钥）
3. 客户端验证证书，生成**预主密钥**，用服务器公钥加密发送
4. 双方用预主密钥派生**会话对称密钥**（AES）
5. 之后通信用**对称加密**（性能原因）

### ✍️ 论文场景

"全链路采用 **TLS 1.3** 传输加密，**AES-256-GCM** 存储加密，密钥由 **KMS** 托管并**90 天轮换一次**，满足等保三级合规。"
