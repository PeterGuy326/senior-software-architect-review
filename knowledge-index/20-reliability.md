# 知识点 20 · 系统可靠性

## 一、核心概念

### 核心指标

| 指标 | 含义 | 公式 |
|---|---|---|
| **MTBF** | 平均无故障时间 | — |
| **MTTR** | 平均修复时间 | — |
| **可用性 A** | Availability | **A = MTBF / (MTBF + MTTR)** |
| **RPO** | 数据恢复点目标 | 容忍丢多少数据 |
| **RTO** | 恢复时间目标 | 容忍多久恢复 |

### SLA 与停机时间对照

| SLA | 年停机 |
|---|---|
| 99% | 3.65 天 |
| 99.9% | 8.76 小时 |
| **99.99%** | **52.6 分钟** |
| **99.999%**（电信级） | **5.26 分钟** |

### 容错技术

| 技术 | 原理 |
|---|---|
| **N 版本程序设计** | 独立开发 N 份 + 多数表决 |
| **恢复块** | 主块失败切备块 |
| **防卫式** | 异常检查、assert |

### 冗余分类

| 类型 | 举例 |
|---|---|
| **结构冗余** | 主备（静态）/ 双机热备（动态） |
| **信息冗余** | 校验码、CRC、海明码 |
| **时间冗余** | 重试、补偿 |

### 可用性战术

1. **错误检测**：心跳、Ping-Echo、异常
2. **错误恢复**：冗余、重试、回滚、降级
3. **错误预防**：事务、进程监视、移除服务

## 三、典型例题

### 📚 选择题 1（可用性计算）

MTBF = 1000h，MTTR = 1h，可用性？
- A. 99%  B. 99.9%  C. **99.9001%**  D. 99.99%

**答案**：C（1000 / 1001 ≈ 0.999）

### 📚 选择题 2（冗余类型）

"CRC 校验"属于哪种冗余？
- A. 结构  B. **信息**  C. 时间  D. 硬件

**答案**：B

### 📚 选择题 3（99.99% 停机）

99.99% 可用性对应年停机时间？
- A. 8.76 小时  B. **52.6 分钟**  C. 5.26 分钟  D. 3.65 天

**答案**：B

### 🎯 案例填空：双机并联可靠度

两台可靠度 R = 0.9 的服务器**并联**，系统可靠度？

**答案**：1 - (1 - 0.9)² = **0.99**

若**串联**？
**答案**：0.9 × 0.9 = **0.81**

### ✍️ 论文场景

"针对 **99.99% 可用性** 要求（年停机 < 52.6 分钟），采用**同城双活 + 异地灾备**（**三地五中心**）架构，实现 **RPO < 5s，RTO < 30s**，**连续 12 个月零 P0 故障**。"
