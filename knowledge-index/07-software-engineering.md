# 知识点 07 · 软件工程

## 一、核心概念

### 生命周期模型

| 模型 | 特点 | 适用 |
|---|---|---|
| **瀑布** | 线性、文档驱动 | 需求稳定 |
| **增量** | 分批交付、每批功能完整 | 需求明确但规模大 |
| **迭代 / 螺旋** | 风险驱动、多轮迭代 | 高风险、需求模糊 |
| **敏捷** / Scrum | 拥抱变化、Sprint 2–4 周 | 互联网 |
| **原型** | 先做原型再开发 | 需求不清 |
| **V 模型** | 瀑布 + 测试对应 | 强调测试 |

### CMMI 5 级

1. 初始级（混乱）
2. **已管理**（项目级可控）
3. **已定义**（组织级标准）
4. **定量管理**
5. **优化**（持续改进）

### ISO/IEC 25010 软件质量 8 特性

1. 功能适用性
2. 性能效率
3. 兼容性
4. **可用性（易用性）**
5. **可靠性**
6. **安全性**
7. **可维护性**
8. **可移植性**

### ATAM 6 质量属性 vs 25010 8 特性

- **ATAM 6 个**：性能、可用性、可修改性、安全性、可测试性、易用性
- **25010 8 个**：上述再加 **功能适用性 + 兼容性 + 可移植性**

## 三、典型例题

### 📚 选择题 1（生命周期选择）

需求非常模糊，风险高，团队规模小，应采用：
- A. 瀑布  B. V 模型  C. **迭代 / 螺旋**  D. 增量

**答案**：C

### 📚 选择题 2（CMMI 级别）

组织已定义标准过程，并在项目间共享经验，处于：
- A. 已管理（2）  B. **已定义（3）**  C. 定量管理（4）  D. 优化（5）

**答案**：B

### 📚 选择题 3（质量特性）

系统从 Linux 迁到 Windows 体现的是哪一质量特性？
- A. 可维护性  B. **可移植性**  C. 可靠性  D. 易用性

**答案**：B

写出三个主要事件：**Sprint Planning / Daily Standup / Sprint Review**
