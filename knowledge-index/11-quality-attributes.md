# 知识点 11 · 质量属性 6 元组 + 战术

## 一、核心概念

### 质量属性场景 6 元组

```
(1) 刺激源 Source      — 谁/什么引发
(2) 刺激  Stimulus     — 发生了什么
(3) 制品  Artifact     — 影响的部分
(4) 环境  Environment  — 运行环境
(5) 响应  Response     — 怎么做
(6) 响应度量 Measure   — 量化指标
```

### 6 大核心质量属性

| 属性 | 核心战术 |
|---|---|
| **性能** | 缓存、并发、资源池、异步、减少计算 |
| **可用性** | **心跳 / Ping-Echo**、冗余（主备/双活）、回滚、重试、例外检测 |
| **安全性** | **认证 / 授权 / 审计 / 加密**、完整性校验 |
| **可修改性** | **模块化、封装、接口抽象、依赖注入**、配置化 |
| **可测试性** | 记录/回放、**接口分离**、内部监控 |
| **易用性** | **撤销 / 取消**、用户启动、系统启动、任务模型 |

## 三、典型例题

### 📚 选择题 1（质量属性战术）

"心跳检测"属于哪种质量属性的战术？
- A. 性能  B. **可用性**  C. 安全性  D. 可修改性

**答案**：B

### 📚 选择题 2（战术匹配）

为提升**可修改性**，应采用：
- A. 缓存  B. **依赖注入**  C. 心跳  D. 加密

**答案**：B

### 🎯 案例填空：场景 6 元组

题目给出："**用户**在**大促时**请求**下单服务**，系统**正常运行**下，**1 秒内返回**，**成功率 > 99.99%**"

| 元素 | 取值 |
|---|---|
| 刺激源 | **用户** |
| 刺激 | **下单请求** |
| 制品 | **下单服务** |
| 环境 | **大促峰值 / 正常运行** |
| 响应 | **完成下单并返回响应** |
| 响应度量 | **RT < 1s，成功率 > 99.99%** |

### ✍️ 论文场景

"识别关键质量属性：**性能（RT P99 < 200ms）**、**可用性（99.99%）**、**可修改性（新支付渠道 3 天接入）**，每个属性均构造 **6 元组**场景作为架构决策输入。"
