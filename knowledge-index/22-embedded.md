# 知识点 22 · 嵌入式 + 实时调度

## 一、核心概念

### 实时系统分类

| 类型 | 特点 | 例子 |
|---|---|---|
| **硬实时** | 违反时限 = 灾难 | 航天、ABS |
| **软实时** | 违反时限 = 降级 | 视频、IoT |
| **弱实时** | 不严格 | 家电 |

### 实时调度算法

| 算法 | 特点 | 调度上界 |
|---|---|---|
| **RMS**（Rate Monotonic） | **静态**，频率越高优先级越高 | n(2^(1/n)-1)，n→∞ 时 **≈ 0.693** |
| **EDF**（Earliest Deadline First） | **动态**，截止期最近优先 | **1.0** |
| **DMS** | 截止期越短优先级越高 | — |

### 可调度性判定

U = Σ(Ci / Ti)，若 U ≤ 上界 则可调度

### 嵌入式软件分层

```
应用层
中间件（文件系统 / 协议栈）
RTOS（FreeRTOS / μC/OS / RT-Thread）
BSP（板级支持包）
HAL（硬件抽象层）
硬件
```

### RTOS 核心机制

- **任务调度**（优先级抢占）
- **信号量 / 互斥量**（同步 + 互斥）
- **消息队列**（任务间通信）
- **中断 + ISR**（中断服务例程）
- **看门狗 WDT**

## 三、典型例题

### 📚 选择题 1（RMS 上界）

RMS 当 n=3 时的利用率上界？
- A. 0.500  B. 0.693  C. **0.779**  D. 1.000

**答案**：C
**解析**：3 × (2^(1/3) - 1) ≈ 3 × 0.2599 ≈ **0.779**

### 📚 选择题 2（可调度判定）

三个任务 (C, T) = (1, 4) (2, 6) (3, 8)，RMS 是否可调度？

**答案**：U = 1/4 + 2/6 + 3/8 = 0.25 + 0.333 + 0.375 = **0.958** > 0.779
**结论**：**RMS 不一定可调度**，但 EDF 可调度（U < 1）

### 📚 选择题 3（RTOS 机制）

避免优先级反转通常采用：
- A. FIFO 队列  B. **优先级继承 / 优先级天花板**  C. 时间片轮转  D. 关中断

**答案**：B

### 🎯 案例填空：嵌入式架构

典型车载控制系统：
- **硬件 → HAL → BSP → FreeRTOS → 中间件（CAN 协议栈）→ 应用**
- 中断响应 < 10μs，关键任务周期 10ms
- 可靠性：**看门狗 + 双机冷/热备 + ECC 内存**

### ✍️ 论文场景

"采用**分层架构**（HAL → BSP → RTOS → 中间件 → 应用），关键任务使用 **RMS 静态优先级**调度，经可调度性分析 **U = 0.65 < 0.693**；通过**看门狗 WDT + 双机热备**保障 **99.999%** 可用。"
